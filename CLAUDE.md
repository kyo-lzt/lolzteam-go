# lolzteam-go

Go API wrapper for Lolzteam Forum (151 endpoints) and Market (115 endpoints).

## Project structure

```
schemas/          — OpenAPI 3.1.0 specs (forum.json, market.json)
cmd/codegen/      — Code generator (reads OpenAPI → emits Go client code)
*.go (runtime)    — HTTP client, retry, rate limiter, proxy, auth, errors
forum_*.go        — Generated Forum client, types, and service files
market_*.go       — Generated Market client, types, and service files
```

## Key commands

- `make generate` — regenerate clients from OpenAPI schemas (`go run ./cmd/codegen`)
- `make lint` — `go vet ./...`
- `make typecheck` — `go build ./...`
- `make test` — `go test ./...`

## Architecture

- **Two separate clients**: `ForumClient` and `MarketClient` (different baseUrl, rate limits)
- **DX pattern**: `client.Threads.List(ctx, params)` — methods grouped by API tags via service structs
- **Codegen**: Go script reads OpenAPI JSON, resolves $refs, groups by tags, emits typed Go files
- **Runtime**: stdlib `net/http`, no external dependencies
- **Generated code is committed** — no codegen needed at install time

## API details

- **Auth**: Bearer token only
- **Forum**: base URL `https://api.lolz.live`, rate limit 300 req/min
- **Market**: base URL `https://api.lzt.market`, rate limit 120 req/min
- **Retry**: 429 (Retry-After), 502, 503 — exponential backoff + jitter
- **Body encoding**: application/x-www-form-urlencoded
- **Parameters**: snake_case preserved (1:1 with API)

## Conventions

- Go 1.23+, stdlib only, zero external dependencies
- Package name: `lolzteam`
- Module: `github.com/kyo-lzt/lolzteam-go`
- All public API uses `context.Context` as first parameter
- Optional params are pointer types with struct tags (`query:`, `form:`)
- Error hierarchy: `LolzteamError` → `HttpError` → `RateLimitError`/`AuthError`/`NotFoundError`/`ServerError` + `NetworkError`
- Generated files have `// Code generated by cmd/codegen. DO NOT EDIT.` header
